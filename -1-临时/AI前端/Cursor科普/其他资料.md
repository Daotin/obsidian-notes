Cursor 三种模式区别

Ask 模式

- 专注于回答问题，不会主动修改代码
- 可以解释代码、提供建议、回答编程相关问题
- 不会自动执行工具调用或修改文件
- 适合获取信息、学习概念、理解代码逻辑

Edit 模式

- 专注于代码编辑和生成
- 可以根据指令修改当前打开的文件
- 不会主动搜索代码库或执行其他工具操作
- 适合快速修改、重写或生成特定代码片段

Agent 模式

- 最强大的模式，可以主动调用工具完成复杂任务
- 能够自动搜索代码库、读取文件、编辑代码
- 可以执行多步骤操作，如创建新文件、修改现有代码、运行命令等
- 适合复杂的开发任务，如构建新功能、调试问题、重构代码

网页搜索功能
Agent 现可自动搜索网页获取最新信息

- 无需显式使用@Web 命令，Agent 可自动判断何时需要搜索网络
- 获取实时更新的信息，提供更准确的回答
- 智能整合网络信息与本地知识

文件忽略功能

- 增强的文件控制功能
- .cursorignore 现在可以阻止文件被添加到聊天或发送进行 Tab 自动完成
- 同时还会忽略这些文件的索引
- 新增 .cursorindexingignore 专门用于控制文件索引

MCP 改进
Project Control Protocol 的重要升级
Agent 现在可以通过 Yolo 模式自动运行 MCP 工具
使用 <project-root>/.cursor/mcp.json 配置项目服务器
支持 Agent 使用 MCP 资源作为上下文
改进的可靠性和 Cursor 中构建 MCP 服务器的生活质量增强
