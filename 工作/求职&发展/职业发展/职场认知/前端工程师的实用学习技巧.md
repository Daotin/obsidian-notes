# 14-前端工程师的实用学习技巧

## 主要

**做**是最好的学！无他。

## 两个工具

- 科学上网
- 单词量

尽量使用英文搜索。会英文的用英文搜，不会英文的转换成英文搜。

## 两个习惯

- 多问工具（搜索，看书，ChatGPT 等）少问人（走投无路才问人，一般问人都是去讨论）
- 多做测试（多动手，多体验）少看文（碎片化文章，可能会分散精力）

1、多问机器少问人

机器：（工具：google，文档，源码，书籍....）

不要为了着急解决掉而去到处问人，厉害的人都是去讨论。

2、多做测试少看文

多自己体验，多动手测试，小到组件，大到产品。不只是看文档，代码运行起来，做一个小 demo 等等

少看文章。少花精力在碎片文上，分散精力，会让自己很难集中精力。

**如果遇到好文章，要慢慢品。**

## 两个技巧

- 跟着大佬走：Follow 大佬 Follow 的 （开阔眼界，看他们在关注什么）

不知道自己应该关注什么，看什么的时候，去看大佬关注的是什么。

- Google
- Github 发现板块—思考和创意
- Medium：国外版比掘金高端 100 倍~~
- MDN：Wiki
- StackOverflow：bug fix

- 核心技巧：**Debug 为王，看源码为皇**

编程的技巧：debug 为第一优先级，看源码是第二优先级。

没有很强的经验判断，就是 debug，可能会到第三方的源码里。网上搜索只是线索，最后还是需要自己 debug。

看源码也是 debug 中的一个能力。
