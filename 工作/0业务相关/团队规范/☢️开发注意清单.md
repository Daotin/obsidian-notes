## 开发中的注意

- [ ] **需要注意**
    - [ ] 数据为空（占位符）
    - [ ] 数据超多
    - [ ] 文字超长（使用`Overflow`组件）
    - [ ] 数字超长（最多容纳多少位数字）
    - [ ] 输入框需要限制长度 `maxlength`
    - [ ] 最小分辨率`1280×768`
    - [ ] Echarts缺省页面
    - [ ] 国际化翻译
        - [ ] 注意中英文切换时，一些渲染的列表要使用computed
        - [ ] 有写页面在切换语言时，需要重新加载接口
    - [ ] 对着原型过一遍，可能有遗漏的地方
- [ ] 表单相关
    - [ ] 点击重置按钮
    - [ ] 标签页切换
        - [ ] 关闭弹窗时，调用`search.resetSearch()`
- [ ] 与接口列表相关
    - **所有接口调用的地方，添加loading**
        - [ ] 页面初始化状态（loading）
        - [ ] 保存按钮loading
        - [ ] 搜索按钮loading
        - [ ] 删除按钮loading
    - **数据增删改后**
        - [ ] 刷新界面后，页码的定位**（除了新增,重置搜索条件,定位到第1页，编辑，删除,保留搜索条件,都定位到当前页）**
    - **假设接口字段异常**
        - [ ] 前端需要有自己的降级措施，防止页面直接白屏、交互异常、甚至挂掉。（使用默认值或者try..catch）
    - [ ] TyTable超出省略
        - [ ] `:show-overflow-tooltip="tableConfigDept.isSingleLine"`
        - [ ] el-link的，需要外部`<div :class="{'table-cell-link':tableConfigDept.isSingleLine}">`
    - [ ] 正确处理页面多状态：
        - [ ] 初始化状态
        - [ ] 加载中
        - [ ] 加载失败
        - [ ] 无数据。
- [ ] 侧边栏相关
    - [ ] 弹框新增后，关闭再打开，数据未清空
    - [ ] 弹框编辑后再新增，数据未清空（使用closed事件`@closed='handleClosed'`）
- [ ] **功能权限（v-auth**和表格列，搜索项）
    - [ ] v-auth权限index,option等相关页面需要同步
    - [ ] 表单不显示机构，产品型号（`const isManager = computed(() => AppStore.isManager)`）
    - [ ] 列表不显示机构，产品型号（`visible: () => isManager`）
    - [ ] **分别用运维和机构账户去登录**使用
- [ ] ==**发现一个问题通常会有一堆相同问题，需要一并修改**==
- [ ] **需要考虑操作不规范**
    - [ ] 不按顺序操作
    - [ ] 操作过快
    - [ ] 考虑弱网情况下可能发生的问题
    - [ ] 反复切换翻页页码
    - [ ] 考虑快速切换菜单、Tab栏可能发生的问题
    - [ ] **页面刷新，导致store的部分数据失效**
    - [ ] **复制url到新标签页，同时操作两个标签页的行为**
    - [ ] 多窗口情况
        - [ ] 打开两个窗口，登录同一个用户A
        - [ ] 打开两个窗口，登录A用户和B用户
- [ ] 其他问题
    - [ ] 考虑上线后可能出现的问题，会不会影响其他的功能？
    - [ ] 考虑任务由点到面。稳定性，兼容性，性能，可扩展性如何?
    - [ ] 不能完全依赖和信任后端字段，导致让页面直接白屏
